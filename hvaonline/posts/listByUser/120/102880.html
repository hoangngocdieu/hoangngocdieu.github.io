Đọc ~ gì bạn mô tả các bước đã làm, thì thấy bạn làm "đúng" rồi nhưng chưa confirm kết quả của cái đúng đó như thế nào.

Tôi mạn phép hướng dẫn bạn cách confirm, bạn phải chăc trạng thái hiện tại của bạn có đúng không theo từng bước.

1. Tạm thời bạn cho phép có thể ping tới router từ internet user -> để bạn sure public IP hiện tại đang dùng là bi nhiêu. Còn Nếu bạn ko muốn thì phải vào router-online status mà xem và note lại.

- Start - Run - CMD - Enter
c:\ftp - Enter
ftp>open public IP (thay public IP = public IP của FTP SRV)
Connected to XXX.YYY.ZZZ.AAA.
220 Serv-U FTP Server v6.4 for WinSock ready...
User (XXX.YYY.ZZZ.AAAnone)):

Tới đây bạn nhập vào user name + pass mà bạn đạ tạo trước đó ở Serv-U.

Mục đích 2 bước trên là muốn check xem bạn NAT port 21 cho FTP SRV bạn đúng chưa? Nếu bạn ko access được thì xem lại cách NAT/public IP. Còn nếu OK, nghĩa là FTP SRV của bạn OK rồi, bây giờ chỉ còn xem cách phan giải domain name (trên no-ip) có phân giải đúng public IP của FTP srv hay ko thôi?

Trong trường hợp của bạn để đơn giản bạn làm theo mô hình sau:

Modem: 192.168.1.1
DHCP start: 192.168.1.100 DHCP End: 192.168.1.200

AP: 192.168.1.2
Disable DHCP

Cable connect như sau: Connect từ port LAN của MODEM tới port LAN của AP.

That's all.

Như vậy client của modem và AP sẽ cùng subnet với nhau (192.168.1.100 ~ 192.168.1.200)/24. Và bạn có thể sharing data với nhau luôn.

Còn nếu bạn muốn client của wire và wireless khác subnet (trên 2 mạng khác nhau - trong trường hợp bạn miêu tả phía trên - 192.168.1.X và 192.168.2.X) thì bạn làm được rồi đó.

Trước tiên bạn phải bỏ chức năng WPA-PSK đi, vì bạn chưa đi net được mà đã lo mã hóa làm gì? Bạn phải chắc khi nào cấu hình AP OK (user wifi có thể đi internet) rồi mới tính đến chuyện mã hóa nha!

Ở đây có 2 options cho bạn phải chọn

1. Sử dụng AP như "wifi switch" nghĩa là bạn chỉ cần set IP cho AP (LAN IP) để sau này bạn connect tới AP để re-config AP mà thôi. Muốn vậy bạn phải sử dụng DHCP của modem mà cấp IP cho toàn bộ LAN (user wire + user wireless). Nhớ khai báo pool IP chừa lại vài IP adress để gán cho AP và spare.

Topology: user wifi -> LAN port AP -> modem -> internet (cùng subnet)

Note: uplink cable -> phải connect LAN của modem tới 1 trong những port LAN của AP (nếu AP của bạn có nhiều port (nhiều port là gồm 1 port internet và X ports LAN))

2. Bạn chia 2 LAN khác nhau: user wire và user wireless khác nhau. Trong trường hợp này bạn phải có AP là loại có nhiều port. Lúc này topology sẽ khác:

user wifi -> INTERNET port AP -> NAT -> modem -> internet (khác subnet)

Note: uplink cable -> phải connect LAN của modem tới internet port của AP. Và bạn phải config IP cho WAN và LAN của AP trên 2 lớp mạng khác nhau.

Tôi đoán bạn sử dụng AP loại 1 port nên bạn hãy chọn option 1. Nếu bạn có AP nhiều port thì tôi sẽ vẽ cho bạn 1 diagram để bạn dễ hình dung mà hiểu được. Tôi sợ viết = text sẽ khó hiểu.
