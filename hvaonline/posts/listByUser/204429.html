Biểu tượng network có hai màn hình màu xanh : bạn có kết nỗi local Area Network (LAN)
Bạn kiểm tra lại bằng cách ping default gateway (thường là 192.168.1.1 hoặc 10.0.0.1) bạn có thể xem default gateway bằng cách vào cmd gõ lệnh ipconfig, nó sẽ hiện ra ở dòng cuối. bạn cố ping xem nó có trả lời không.

nếu ping thấy default gateway, bạn tiến hành kiểm tra ip của bạn có cùng lớp mạng với default gateway hay không (nếu ip của bạn có 3 số đầu giống với default gateway thì cùng lớp mạng và ngược lại)

nếu ip của bạn và default gateway đã cùng lớp mạng rồi mà vẫn không vào được net bạn tiến hành kiểm tra dns server, bạn vào cmd gõ lệnh nslookup, nếu nó thông báo tên và ip của server thì dns bình thường, ngược lại bạn vào phần cấu hình ip đặt lại dns server theo kiểu tự động (automaticaly) hoặc sử dụng dns server do bạn chỉ định( của google chẳng hạn 8.8.8.8)

nếu dns server hoạt động bình thường bạn tiến hành ping thử một host nào đó để kiểm tra đường truyền.

Cái bạn đang nói đến gọi là Multi-Master đó, sau khi cấu hình thành công thì mọi thông tin trong AD đều được cập nhật giữa hai máy chủ, nếu muốn bạn có thể đồng bộ Data giữa hai máy. khi sự cố xảy ra với một trong hai máy thì máy kia có thể thay thế để phục vụ user.

Mình chưa thử đồng bộ Data nhưng AD thì tốt.

Để thực hiện:

1> Chuẩn bị: ngoài máy chủ đang dùng bạn cần có một máy chủ thứ 2 (tốt nhất là chạy cùng phiên bản hệ điều hành với máy có sẵn).
2> Cấu hình:
-Ở máy chủ có sẵn, bạn tích hợp DNS vào AD: vào DNS manager, click phải vào domain, chọn Properties (dễ tìm lắm vì chỉ có 1 dòng có properties thôi), sau đó ở tab general, có 3 nút "Change" ở phía trên bên phải, bạn click vào cái thứ 2, sau đó tick chọn cái checkbox, ok.
-Ở máy thứ 2, bạn cài DNS nhưng không cấu hình.
-Join máy thứ 2 vào domain (tốt nhất là sử dụng user administrator hoặc tương đương).
-Nâng máy thứ 2 lên thành DC nhưng không chọn dòng primary mà chọn dòng addtition .

Thế là đã ok rôi. để đồng bộ Data bạn tham khảo thêm từ nguồn khác, chắc cũng dễ, mình lười đọc quá!.
