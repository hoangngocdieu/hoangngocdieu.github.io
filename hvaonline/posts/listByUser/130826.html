Em tạo 1 server host hơn chục site trên IIS. Sau khi đã thay đổi user và AppPool ID cho từng website và chạy ngon lành thì hôm nay vào lại thấy có vấn đề khi duyệt site, báo lỗi 401.3 Unauthorized: Access is denied due to an ACL set on the requested resource.
Các user cho từng site em cho vào group Webhosting, các AppPool ID cho vào group IIS-WPG. Đã phân quyền đầy đủ cho các user này trên các thư mục gốc. Dùng IIS Diagonostics check permission của các user thì thấy báo lỗi ko đủ quyền. Log on vào computer với acc trên thì thấy vẫn tương tác đầy đủ. Một số site ko chạy thì làm y hệt các site đã chạy về user và permission toàn bị lỗi 401.3 ở trên.

#!/usr/local/bin/perl
# file:tcp_echo_cli1.pl
# usage:tcp_echo_cli1.pl [host] [port]
# echo client,tcp version
use strict;
use Socket;
use IO::Handle;
my ($byte_out,$byte_in)=(0,0);//declare globals,create 2 global variables for keeping track of the number of bytes we send and receive.
my $host =shift || 'localhost' ;
my $port =shift ||getservbyname('echo','tcp');
my $protocol = getprotobyname('tcp') ;
$host =inet_aton($host) or die "$host :unknown host";
socket (SOCK,AF_INET,SOCK_STREAM,$protocol) or die "socket() failed:$!";
my $dest_addr=sockaddr_in($port,$host)
connect (SOCK,$dest_addr) or die "connect() failed: $!";
SOCK ->autoflush(1);
while (my $msg_out=<>{
print SOCK $msg_out;
my $msg_in=<SOCK>;
print $msg_in;
$bytes_out +=length($msg_out);
$bytes_in +=length($msg_in);
}
close SOCK
print STDERR "bytes_sent=$bytes_out,bytes_received=$bytes_in\n";
