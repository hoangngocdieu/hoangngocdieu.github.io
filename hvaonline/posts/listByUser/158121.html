Chào mọi người hiện tại nhóm em đang phát triển một ứng dụng web chạy trên Window Server 2008.
Em dùng IIS làm web server và dùng Oracle database.

Vấn đề của em là không biết nên config server như thế nào là hợp lý:
Yêu cầu ứng dụng: tối đa khoảng 10,000 connection cùng lúc và phục vụ hàng chục ngàn lượt truy cập mỗi ngày ( hiện tai em ước tính vậy thôi chứ có khả năng nhiều hơn).
+Trang web phục vụ về tìm kiếm nên request datatabase khá nhiều.
+Có hỗ trợ Upload/Download file (dung lượng chỉ tầm ~10Mb/file ,tỉ lệ Down file là 40% cho 1 lượt truy cập, Up file là 10%)
Câu hỏi đặt ra:
1. Có nên cài đặt Database , Web Server và File server trên cùng 1 PC.
2. Mô hình Load balancing cho Database, Web Server và File Server như thế nào là hợp lý.

Mong mọi người cho em chút giải pháp hoặc hướng đi.
Em là sinh viên nên kinh nghiệm không nhiều. Mọi người thông cảm cho nếu như mấy câu hỏi của em quá dở hơi (hay đại loại như vậy ^^).

Cảm ơn mọi người đã quan tâm.
