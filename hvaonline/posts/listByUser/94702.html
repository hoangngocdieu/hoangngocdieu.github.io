Mạng bên mình như sau: 1 web application server -- 1 secure server. Qui tắc các máy clients muốn vào web applicatrion phải qua chứng thực ( sử dụng RSA SecurID ). Khi user kết nối web application phải kết nối qua VPN để chứng thực. Vấn đề rắc rối là sau khi chứng thực, các kết nối internet từ máy clients kg còn đi trực tiếp đến ISP mà phải đi qua con secure server trước- lúc này khi check IP các clients thì nó mang IP của con secure server. Vấn đề chuối vô cùng là các clients lại nằm ở nước ngoài, thành ra các request của clients khi đó phải đi qua VN(secure server đặt tại VN ) rồi mới trở ra lại bên ngoài. Mà đường truyền VN thì chuối thế nào , chắc kg nói ra thì ai cũng biết. Nếu một vài clients vô VPN xong rồi coi phim online hay load cái gì nặng nặng chút thì con server secure chết là cái chắc ( lúc đó mạng lag vô cùng ).

Các bác có ai có tài liệu hướng dẫn config server VPN sử dụng token RSA SecurID kg?
Vấn đề là hiện nay công ty mới mình đang dùng RSA SecurID nhưng khi kết nối VPN thì tất cả các kết nối internet khác đều phải sử dụng qua server VPN này ( khi kết nối VPN, các clients sẽ mang IP của VPN Server ). Do vậy các kết nối internet sẽ phụ thuộc vào con server secure này, nếu một vài bác nào đó vô đó xem phim online hoặc download gì gì đó thì mạng internet sẽ chuối ngay.
