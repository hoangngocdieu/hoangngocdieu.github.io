Không hiểu sao mình config https cho Apache rồi, sau đó mình enable site SSL để nối mod_jk đó với Tomcat

Khi vào tomcat, chỉ cần gõ domain là vào ok (abc.com.vn)

Khi vào bằng https, cần get certificate để vào. Đúng ra thì phải vào thẳng tomcat (liferay), nhưng nó lại nhảy vào Apache...

Đúng thật là điên đầu với thằng này luôn, lỗi ko ra lỗi mà cứ nhảy vào Apache hoài...

đây là file config site abc.com.vn trong site available, sau đó a2ensite sang site-enabled

<VirtualHost *:80>

ServerAdmin [email]admin@myqgs.com[/email]
ServerName abc.com.vn
ServerAlias *.myqgs.com

DocumentRoot /var/www

JkMount /* liferay

JkUnMount /PrivacyPolicy.htm liferay

JkUnMount /ServerLink.txt liferay

JkUnMount /Setting.plist liferay

JkUnMount /TermsOfUse.htm liferay

JkUnMount /Account.plist liferay

DirectoryIndex index.jsp index.html

</VirtualHost>
 


abc.com.vn-ssl
<IfModule mod_ssl.c>
<VirtualHost *:443>


ServerName abc.com.vn

DocumentRoot /var/www

SSLEngine on

SSLCertificateFile /etc/apache2/ssl/server.crt
SSLCertificateKeyFile /etc/apache2/ssl/server.key

# ...
# (the config file with your distribution will probably have
# a sensible set of options for SSL as well.)

JkMount /* liferay

JkUnMount /facebook liferay

JkUnMount /facebook/* liferay

JkUnMount /spa liferay

JkUnMount /spa/* liferay

JkUnMount /nails liferay

JkUnMount /nails/* liferay

JkUnMount /openx liferay

JkUnMount /openx/* liferay

DirectoryIndex index.jsp index.html



</VirtualHost>
