Chào bạn,
Để máy A 192.168.1.10/24 thấy máy B:192.168.4.10/24 thì ta tiến hành cấu hình routing như sau:
1.Máy A add default route(gateway) 192.168.1.1
2.Máy B add default route(gateway) 192.168.4.1
3.Cấu hình routing dynamic RIP cho 3 router R1,R2,R3

a.Config Rip R1

R1>en
R1#conf t
R1(config)#router rip
R1(config-router)#net 192.168.1.0 255.255.255.0
R1(config-router)#net 192.168.2.0 255.255.255.0
R1(config)#do wr

b.Config Rip R2

R2>en
R2#conf t
R2(config)#router rip
R2(config-router)#net 192.168.2.0 255.255.255.0
R2(config-router)#net 192.168.3.0 255.255.255.0
R2(config)#do wr

c.Config Rip R3

R3>en
R3#conf t
R3(config)#router rip
R3(config-router)#net 192.168.3.0 255.255.255.0
R3(config-router)#net 192.168.4.0 255.255.255.0
R3(config)#do wr

Done

Để làm được việc này thì theo mình, bạn cần phải làm 2 vấn đề chính như sau :
Thí dụ bạn có domain abc.com. Trước tiên bạn vào Cpanel Domain, bạn chuyển A Host Record về IP của bạn, ví dụ : *.abc.com -> A Record
Công việc này để làm gì ? Đơn giản là từ nay về sau, bất cứ subdomain nào được gõ vào adress bar thì cũng trỏ về đúng IP của bạn, chẵng hạn như : www1.abc.com, blog.abc.com .v..v... sau đó thì đến công việc còn lại là làm sao biết subdomain nào là của host nào, nếu bạn dùng Apache làm webserver thì chuyện đơn giản là bạn vào Add Virtual Host cho từng subdomain là xong !
Tài liệu về Virtual Host thì nghiên cứu ở đây : http://httpd.apache.org/docs/2.0/vhosts/
Sources để quản lý các subdomain, giống như trang kiss.vn mà bạn đã đề cập thì vào đây download và nhớ đọc document để biết cách dùng nhé : http://www.milliscripts.com/
