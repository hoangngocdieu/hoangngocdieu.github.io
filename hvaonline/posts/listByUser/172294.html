Bạn thử cấu hình file smb.conf như thế này xem:
Code:
[global]
workgroup = WORKGROUP
server string = samba file server
security = user
[TaiLieuLinux]
path = /media/DATA/TaiLieuLinux/
public = yes
writeable = yes
browseable = yes
[Pictures]
path = /home/duongit/Pictures/
public = yes
writeable = yes
browseable = yes


Sau đó bạn chạy lệnh:
Code:
sudo smbpasswd -a <tên user của bạn>

Nhập password rồi restart lại samba daemon.

Lúc mới nối mạng, chưa bị arp attack đấy, bạn vào cmd gõ lệnh:
Code:
arp -a

Lúc đó bạn sẽ xác định được địa chỉ Mac của gateway (thông thường ip của gateway là 192.168.1.1) trước khi bị arp spoof
sau đó bạn gõ lệnh:
Code:
arp -s [địa chỉ ip của gateway] [địa chỉ MAC của gateway]

Ví dụ:
Code:
arp -s 192.168.1.1 00-AA-00-4F-2A-9C

Mình nhớ là HVA có mấy bài về cái này rồi, bạn search thử xem sao. Từ khóa là arp spoof, arp poision.
