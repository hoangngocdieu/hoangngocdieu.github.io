_VoT_ wrote:
Xin hỏi: Với người mới nhập môn CSDL thì nên học SQL Sever 2000 hay SQL Sever 2005 ? 


Theo tôi, trước khi bro muốn so sánh hai phiên bản phần mềm thì bro cần cài đặt nó trước vậy .
SQL Server 2005 nó đến 5 phiên bản lận, mổi phiên bản nó đều có đặc tính khác nhau .
Phần mềm SQL Server 2005 rất đắt tiền, nếu bro không có money thì bro nên sử dụng phiên bản Express Edition, nếu bro mua ở các cửa hàng CD thì có thể sử dụng phiên bản Enterprise (nếu bro cài đặt trên Windows XP với phiên bản này thì có thể bị lổi trong quá trình cài đặt) .
- So với phiên bản củ SQL 7.0, SQL 2000 thì SQL 2005 chú trọng đến công tác bảo mật nhiều hơn .
- Mở rộng một số chức năng so với phiên bản trước ví dụ như Report, SQL Server Profiler , Schema, thêm một số kiểu data mới, sao lưu dử liệu nhanh chóng, an toàn hơn... nhằm đem lại cho người sử dụng thuận tiện, nhanh chóng .
- Chức năng SSL được tự động hóa hơn, thay vì lúc trước trên MS SQL 2000 khai báo tùm lum, thì trên SQL 2005 nó khai báo cho mình .... tự động luôn .

Nếu bạn sử dụng SQL 2000 quen, khi bạn chuyển sang SQL Server 2005 có thể bạn cảm thấy lạ vì một số chức năng trong SQL Server 2000 như Enterprise Manager đều ... biến mất thay thế vào nó là SQL Server Management Studio .

Hầu như cái nào mang tính lưu trử thì có thể được gọi là CSDL (tạm dịch Database thành CSDL) ví dụ như tôi thấy bài viết của tay nào đó trên HVA hay quá, tôi lưu nó lại trong notepad thì nó được gọi là CSDL !

SQL có mấy câu lệnh mà thôi SELECT (đọc, xem) , UPDATE (cập nhật) , DELETE (xóa), INSERT (thêm) ở mổi câu lệnh thì xem xét nó đi theo với mệnh đề, hàm nào và ... học thuộc lòng nó .

Sau đó thiết kế 1 database , rồi mở SQL Query Analyzer ra rồi từ đó xem nó làm việc như thế nào . Sách bọng tham khảo thì có rất nhiều trên Internet .
