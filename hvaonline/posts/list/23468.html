Hi các bạn,

Hiện tại mình đang làm một Home server ở nhà , mình sử dụng ADSL của Softbank (Japan). Các thông tin về home server của mình như sau :
Mình đăng ký một domain riêng tại Onamae.com
Mình đăng ký dịch vụ Dynamic DNS của Zoneedit và EveryDNS
Và đã thiết định Nameserver cho domain của mình rồi.

Sử dụng các dịch vụ tại http://www.websitepulse.com/help/tools.php thì test thấy web , dns chạy tốt. Coi log của Apache có thấy địa chỉ bên ngoài access vào.

Nhưng nếu mình dùng Mozilla hoặc IE ngay chính trên server để truy cập vào thì lại không được, chỉ ra 1 trang trắng nhách, và xem log thì chẳng thấy thông tin gì cả.

Không biết trường hợp này bị như thế nào.

Hi Choen va LinuXpert

Mình là khoaitu đây, không biết sao bị lock nên mình đăng ký cái nick khác để vào.

Choen nói : ngồi trên chính máy server thì không truy cập được. Chắc có thể đúng, nhưng mình chạy qua máy khác, cũng dùng một Lan luôn, cũng không truy cập được.

Để mình mô tả cái Lan và Server của mình nhé.

Server : Fedora 9
Lan thì thế này

Có 2 máy , 1 server, 1 client
IP Server : 192.168.3.3
IP client : 192.168.3.2
Default getway : 192.168.3.1

Modem đã được cấu hình NAT đầy đủ, vì từ trang web check web site, check DNS có thể truy cập vào máy server được.

IP của nhà cung cấp ADSL, vi dụ là 219.8.224.118
Domain mình đăng ký là : aaa.com

Khi mình ngồi trên máy server :
>ping aaa.com thì trả về 219.8.224.118
Vậy là ping tốt.

Nếu ping từ client thì cũng trả về IP 219.8.224.118

>nslookup aaa.com
Server:local.gateway
Address : 192.168.3.1

Non-authorities answer
Name : aaa.com
Address: 219.8.224.118

Vậy cũng đâu có vấn đề gì

Nếu dùng Mozilla ở Server truy cập localhost hoặc 192.168.3.3 thì OK nhưng gõ tên domain vào thì ra trang trắng nhách. Tương tự khi truy cập từ client cũng vậy.

Mình cũng chưa tìm ra được nguyên nhân,nếu dùng Dynamic DNS thì đâu cần config DNS,BIND ở trên server phải không các cao thủ.

Mong các cao thủ trả lời dùm mình nhé, rất nhức đầu.
Thanks
