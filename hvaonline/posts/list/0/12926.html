Tớ lập ra topic này để ai có vướng mắc gì khi dùng Fedora, post lên đây để anh em cùng tìm cách khắc phục (Bản 7 thì gọi là Fedora, còn các bản trước là Fedora Core)

Phần lớn anh chị em khi mới làm quen với Linux đề cài 2 OS trên máy của mình: Windows và một Distro nào đó của Linux

Tớ xin mở màn với vài lỗi mà tớ đã từng gặp, cấu trúc là anh em sẽ đưa ra chính xác thông báo lỗi, dự đoán nguyên nhân và đưa ra cách khắc phục

1. Bạn đã cài Windows trước, rồi cài Fedora. Nhưng sau này, bạn lại có ý định chia lại partition và bạn vào Wins để làm điều này, thế nhưng sau khi chia lại xong, boot vào Fedora thì bạn gặp lỗi sau:

root (hd0,3)
Filesystem type unknown, partion type 0xf Kernel /boot/vmlinuz-2.6.21-1.3194-FC7 ro root=LABEL=/1 rhgb quiet

Error 17: Cannot mount selected partition

No kernel packages were installed on your system. Your boot loader config will not be changed
 

Nguyên nhân: Lỗi này xảy ra chính là do bạn chia lại partition và phân vùng cài Fedora đã được đánh số lại nên hệ thống không thể tìm thấy kernel package trên phân vùng bị đánh số sai

Khắc phục: Hãy boot vào rescue mode của Fedora, dùng lệnh fdisk -l để xem chính xác phân vùng nào bạn đã cài Fedora lên đó, sau đó sửa lại trong /etc/grub.conf dòng root (hd0,x) cho tương ứng

2. Sau khi bạn upgraded hoặc cài lại Wins sau khi cài Fedora hoặc làm gì đó liên quan đến partition, khi boot vào Fedora bạn gặp lỗi sau:

[Minimal BASH-like line editiing is supported. For the first word, TAB
lists possible command completion. Anywhere else TAB lists the possible completions of a device/filename. ]

grub>
 

Lỗi này là do grub của bạn đã bị hỏng, khắc phục như sau:
Xác định xem phân vùng /boot của bạn nằm ở đâu, bằng cách:
- Từ dấu nhắc grub>_ trên, bạn gõ find /boot/grub/stage1
- hoặc boot vào chế độ rescue mode (trong F7 thì có luôn menu, còn trong FC6 trở về trước, bạn gõ linux rescue) và gõ fdisk -l để xác định phân vùng /boot, ký hiệu là /dev/hdax
+ Từ chế độ rescue gõ grub, sau đó gõ
root (hd0,x-1) - (do các partition được đánh số từ 0)
setup (hd0) - nếu bạn muốn cài Grub lên MBR
setup (hd0,x-1) - nếu muốn cài Grub lên /boot partition
quit 


3. Cũng liên quan đến partitioning, bạn dùng đĩa boot để chia lại, bạn để ý và nhìn thấy label của / (the root filesystem) là "/1", tự nhiên bạn có ý định đổi lại nó, thế là khi boot vào Fedora bạn sẽ gặp lỗi tương tự như:

Uncompressing Linux... Ok, booting the kernel.
Red Hat nash version 6.0.9 starting
mkrootdev: label /1 not found
mount: error 2 mounting ext3
mount: error 2 mounting none
switchroot: mount failed: 22
umount /initrd/dev failed: 2
Kernel panic - not suncing: Attempted to kill init!
 

Khắc phục: hãy đổi lại label cho / partition như cũ: /1

4. Bạn load bản Fedora 7 về và cài đặt "ngon lành" nhưng khi bạn tắt máy bằng:
Code:
/sbin/shutdown -h now

hoặc tắt từ menu: System --> Shutdown
thì đều không được, hệ thống stop tất cả các services xong rồi đưa ra một thông báo

System halted...
 

và treo luôn ở đó. Bạn phải nhấn nút Power "lỳ" để tắt hẳn.

Nguyên nhân có thể là do: Fedora 7 đã mặc định enable ACPI (Advanced Configuration and Power Interface) mà không dùng APM (Advanced Power Management). Bạn hãy thử vài solution sau xem có được không:
log vào với root permission và gõ: Code:
/sbin/poweroff
 

Trong /boot/grub/grub.conf, tìm dòng bắt đầu bằng kernel và thêm vào đoạn
Code:
acpi=off apm=on apm=power-off

Code:
hoặc acpi=force

 

Check xem acpi đã được enabled chưa bằng lệnh
Code:
chkconfig --list | grep acpi

 

gedit /etc/rc.d/init.d/halt
tìm đoạn này:
Code:
case "$0" in
   *halt)
	message=$"Halting system..."
	command="/sbin/halt"
	;;

sửa thành:
case "$0" in
*halt)
message=$"Halting system..."
command="/sbin/halt -p"
;;

6. Một ngày đẹp trời nào đó, sau khi gõ username và password để logon thì bạn nhận được thông báo:
User’s $HOME/.dmrc” file is being ignored. This prevents the default session and language from being saved. File should be owned by user and have 644 permission. User $HOME directory must be owned by user and not writable by other users 

Đúng như câu thông báo, và bạn đã google, bạn thử:
+ Tìm tất cả các file có tên là ".dmrc"
Code:
find / -name ".dmrc" -print

sau đó, đổi permission thành 644
Code:
chmod 644 every_file

rồi thay quyền sở hữu của các file đó
Code:
chown your_login_name /home/your_login_name/.dmrc

Sau đó, bạn thử logon lại nhưng vẫn gặp thông báo lỗi trên.

Đừng tức giận, hãy thử solution sau:
+ Vẫn làm 3 bước trên
+ Thêm 1 bước: Gán lại quyền để chỉ cho phép bạn truy cập trên thư mục /home/your_login_name
Code:
chmod 700 /home/your_login_name

Ctrl + Alt + Backspace để thấy kết quả :lol
