Hi every one
Mình vừa mới cài thằng Linux CentOS 5 xong. Nhưng gặp 1 trường hợp rất khó hiểu, các bạn có thể giúp mình ko?
Máy mình chia Partition như sau:
C:\ Window XP Home Edition (có bản quyền)
D:\Data
E:\Swap (1GB) do RAM mình 512
F:\CentOS 5 (5GB).
1./ Sự cố về lệnh mount
Nhưng khi mình vào CentOS 5 thì muốn mount ổ C thì nó ko phải là hda1 hay hda0 mà là sda1. Nhưng chỉ mount được ổ C, còn ổ D dùng sda2 thì ko được.
2./ Sự cố về CONFIG BIND
Mình đã từng cấu hình wa Red Hat Linux 9 và Fedora Core 4.
Thông thường thì file cấu hình sẽ là /etc/named.conf <- đối với RH 9
Còn FC 4 thì do có chroot nên sẽ nằm ở đây /var/named/chroot/etc/named.conf đối với bạn nào cài gói có chroot. Và sẽ nằm ở đây /etc/named.caching-nameserver.conf đối với bạn nào cài gói caching-nameserver. Tài liệu tham khảo http://www.centos.org/docs/5/html/De...S/ch-bind.html

Nhưng khỗ nỗi mình tìm theo 2 đường dẫn trên cũng ko tìm ra cái file named.conf để cấu hình DNS.

Và điều chắc chắn là mình đã cài gói này trong phần cài đặt CentOS 5. Dùng lệnh "rpm -qi bind" version 9.3.3, release 7.el5.

Bạn nào biết file named.conf ở đâu, xin chỉ cho mình với.
Thân

1/ hda hay sda cũng không sao cả. Bạn post cái lỗi mà nó thông báo khi mount lên xem.

2/ Sau khi cài Bind, nó có các file example cho mình, nên:
- Copy tất cả file nằm trong /usr/share/doc/bind-9.3.3/sample/etc vào /etc
- Copy tất cả file nằm trong /usr/share/doc/bind-9.3.3/sample/var/named vào /var/named

Có thể đường dẫn đến những file example trong hệ thống của bạn sẽ khác, bạn dùng lệnh sau để tìm nó:
$find / -name named.conf

Nên thử
rpm -ql bind<version x y x> để tìm cách path named.conf nằm ở đâu

không thì:
find / -depth -type f -name "named.conf" -print

Riêng cái vụ "ổ" thì trên Linux không có khái niệm ổ C: hay ổ D: gì cả. Muốn mount thì mout partition và mount cho đúng partition type là được.

Làm theo lời các bác xong, em vẫn ko tài nào tìm được cái thằng "named.conf".
Nhưng đúng theo @nil nói là nó copy từ source từ
/usr/share/doc/bind-9.3.3/sample/etc vào /etc thiệt.

Vì lúc đầu em tìm nó chỉ ra file "named.conf" là ở đường dẫn trên (dùng lệnh locate named.conf). Mặc dù biết nó là file DOCUMENT nhưng vẫn config thử. Nhưng khi Start servrice thì báo lỗi file /etc/named.root.hints not found.

Wái thiệt. lò mỏ lên internet kiếm tiếp và post bài hỏi thăm tiếp. cuối cùng tìm ra cái link này, của 1 anh bạn bên Forum NhấtNghệ đưa.
http://www.centos.org/modules/newbb/...=8165&forum=41
Làm theo sao khi update xong, đúng là các file mình cần cấu hình hiện ra rõ mồm một. Vui mừng 2 phút, cấu hình zone file thuận và zone file nghich xong, restart lại Service named, tiếp tục lỗi
/etc/named.root.hints not found.
Mà wả thật khi find cũng ko thấy file này, có fải em cấu hình sai cái gì ko ta.
VÌ nhớ ở cái DISTRIBUTION # thì là file "named.ca" (gồm địa chỉ 13 ROOT DNS server) chứ ko fải cái file này.

Mình xài CentOS, cũng gặp tình trạng như trên. Khi cài Bind lên rồi

Code:
[root@CentOS CentOS]# rpm -qa | grep bind
bind-utils-9.3.3-10.el5
ypbind-1.19-7.el5
bind-libs-9.3.3-10.el5
bind-9.3.3-10.el5


thì ko tìm thấy file cấu hình

Code:
[root@CentOS CentOS]# find / -name "named.conf"
/etc/dbus-1/system.d/named.conf
/etc/named.conf
/usr/share/doc/bind-9.3.3/sample/etc/named.conf
/usr/share/logwatch/default.conf/services/named.conf



[root@CentOS CentOS]# find / -name "named.root"
/usr/share/doc/bind-9.3.3/sample/var/named/named.root

[root@CentOS CentOS]# find / -name "named.root.hints" -print
/usr/share/doc/bind-9.3.3/sample/etc/named.root.hints


Mọi người giải thích giúp mình tại sao khi cài lên rồi nhưng ko thấy file cấu hình được ko?

Ngoài cách copy file cấu hình từ sample sau đó vi lại thì còn cách giải quyết nào khác nữa không?
