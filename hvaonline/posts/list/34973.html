Hôm nay, xin được chỉ dẫn cho làm sao để publish website của công ty tôi và ở ngoài internet có thể truy cập được vào:

- Network : Mạng cáp quang của VNPT qua 1 converter + router Linksys WRTG54L 4.30.7

- IP tĩnh : 113.161.128.xxx

- Server Linux : Stand-alone, Fedora 12 x86

- Mục đích : publish website của công ty, giới thiệu sản phẩm, dịch vụ.

Vấn đề :

Website công ty tôi không thể truy cập từ ngoài internet vào mà chỉ có thể truy cập nội bộ.

- Đã NAT port 80 / tạm thời tắt Iptables để test.
- Công cụ dùng để check port : canyouseeme.org và yougetsignal.com --> nhưng đều báo :

Error: I could not see your service on 113.161.128.xxx on port (80)
Reason: Connection timed out

hoặc

Port 80 is closed on 113.161.128.xxx.

Rất mong được chỉ dẫn để tôi có thể thực hiện việc publish webserver?

Nếu web server ở bên trong LAN chạy vẫn bình thường (vào web vẫn bình thường) thì kiểm tra lại trên Router Linksys xem port forwarding đúng chưa?

Để publish 1 cái webserver ra ngoài Internet với mô hình hiện tại của bạn thì rất đơn giản.
Việc thứ 1: Đảm bảo web server chạy trên port 80. Cái này test ngay trong LAN được.
Việc thứ 2: Cấu hình trên Linksys cho nó forwarding port 80 vào con web server này.

mình đã tiến hành việc của bạn nói
Việc thứ 1: Đảm bảo web server chạy trên port 80. Cái này test ngay trong LAN được.


có thể kết nối bằng địa chỉ : www.domain.vn từ trong LAN

lệnh : netstat -an | grep :80
--> kết quả:
tcp 0 0 192.168.20.2:80 0.0.0.0:* LISTEN


Việc thứ 2: Cấu hình trên Linksys cho nó forwarding port 80 vào con web server này.

Đã configure Port Range Forward cho cổng 80 cho địa chỉ IP : 192.168.20.2

nhưng không thể connect tới địa chỉ internet : 113.161.128.xxx --> vui lòng chỉ dẫn
