Hi!
Mình đang cài đặt VirtualBox, Xen, OpenVZ trên nền CentOS 5.
Mình gặp 1 số sự cố như sau:

1./ Đôí vơí Virtual Box, mình down load bản rpm redhat enterprise 5 cho CentOS 5
Quá trình cài đặt Virtual Box ko gặp vấn đề gì, nhưng khi mình chạy máy ảo, thì gặp thông báo thiếu kernel module cho virtualbox.
Mình chạy lệnh này:
# /etc/init.d/vboxdrv setup
Báo lỗi như sau:

Warning: using /usr/src/linux as the source directory of your Linux kernel. If this is not correct, specify KERN_DIR=<directory> and run Make again.
 

Mình vào file Makefile của Vbox1.6.2 để sửa lại dòng lệnh sau

KERN_DIR := /usr/src/vboxdrv-1.6.2/linux 

sau đó mình chạy lại lệnh
# /etc/init.d/vboxdrv setup
tiếp tục báo lỗi như sau:
Makefile:154: *** Error: unable to find the include directory for your current Linux kernel. Specify KERN_INCL=<directory> and run Make again. Stop.
 


Đến đây mình không biết KERN_INCL chỉ ra trong thự mục nào? Và search trên google thì không thấy hướng dẫn fix dành cho CentOS 5, chỉ có Debian và Gento, Ubuntu.
Bạn nào từng làm qua, xin vui lòng hướng dẫn giúp mình.

2./ Sau khi thất bại với Virtual Box, mình đã cài thử Xen, và OpenVZ trên link http://www.server-world.info/en/note?os=ce5&p=virtual
Đôí vơí Xen, khi mình Install Xen xong, gặp sự cố ngay phần Install GuestOS, do rebood máy lại mình không thấy phân vùng ảo là Xen để cài đặt Guest OS vào.
Đôí vơí OpenVZ, ngay khi sưả file /boot/grub/grub.conf
Mình không thấy fân vùng giống như WEB là
Code:
kernel /vmlinuz-2.6.18-8.el5.028stab031.1 ro root=/dev/VZ/Host00 console=tty0

initrd /initrd-2.6.18-8.el5.028stab031.1.img

Mình nghĩ do định nghĩa ổ cúng của máy mình và máy hướng dẫn khác nhau. Phân vùng của mình định nghĩa trong file grub.conf như sau:

Code:
kernel /boot/vmlinuz-2.6.18-53.1.19.el5.028stab053.14 ro root=LABEL=/ rhgb quiet
initrd /boot/initrd-2.6.18-53.1.19.el5.028stab053.14.img


Mình tiếp tục là theo cho đến bước
# /etc/rc.d/init.d/vz start nhưng ko start được
Mình đã # tail /var/log/message

Aug 1 04:27:18 localhost hcid[2398]: Default passkey agent (:1.6, /org/bluez/applet) registered
Aug 1 04:27:18 localhost pcscd: winscard.c:219:SCardConnect() Reader E-Gate 0 0 Not Found
Aug 1 04:27:18 localhost last message repeated 4 times
Aug 1 04:27:21 localhost setroubleshoot: 2008-08-01 04:27:21,284 [rpc.ERROR] attempt to open server connection failed: (2, 'No such file or directory')
Aug 1 04:43:28 localhost Installed: ovzkernel.i686 2.6.18-53.1.19.el5.028stab053.14
Aug 1 07:16:38 localhost Installed: vzyum.noarch 2.4.0-11
 

Đến đây mình hoàn toàn không biết phải tìm kiếm thêm những gì nưã để có thể cài được 1 hệ điều hành ảo trên nền CentOS.
Ai đã làm qua rôì, hay dùng những soft khác hay hơn, vui lòng hướng dẫn giúp mình.
Lưu ý: CentOS 5 đã cài RAID 1 (raid mềm của CentOS 5) trước khi tiến hành SETUP máy ảo
