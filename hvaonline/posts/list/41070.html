Tốt hơn hết bạn cần nắm vững một số ngôn ngữ lập trình cơ bản như C và hướng đối tượng như C++, java ... như thế lúc lập trình game hiểu bản chất hơn.

Mới đầu làm quen bạn cần xác định xem mình sẽ viết game dùng engine gì để hỗ trợ.
Kì vừa rồi, mình được học môn Đồ hoạ và hiện thực ảo ở trường ĐH, nhóm mình đã chọn engine Unity để lập trình một game 3D góc nhìn người thứ nhất.
Bạn có thể tham khảo trên trang chủ
http://unity3d.com/

Mình thấy engine này rất mạnh và bạn cũng không cần code quá nhiều. Chi tiết bạn tham khảo tại trang home của Unity. Bản free cũng đã khá mạnh rồi: ví dụ như đối tượng First Person Controller chỉ cần kéo thả vào là bạn đã có thể điều khiển và dùng chuột để nhìn chung quanh như trong Counter Strike..., còn nếu tìm trên các diễn đàn sẽ thấy bản Full, hỗ trợ đổ bóng mặt trời, hiệu ứng hình ảnh cao cấp........

Code với engine này cũng khá thoáng: bạn có thể code bằng javascript, C#, hày thậm chí php; thường thì code bằng javascript.

Bạn nên học theo kiểu vừa xem tutorial vừa làm theo như nhóm mình thì sẽ hiểu rất nhanh .
Tutorial ở đây
http://www.vtc.com/products/Introduction-to-Game-Development-Using-Unity-3D-Tutorials.htm

Mình xin tóm tắt lại những gì các bạn trên nói, để lập trình game bạn cần có những kiến thức sau, hoặc có một nhóm giỏi những kiến thức sau.

1.Lập trình.
2.Toán học.
3.Vật lý trong game dùng để tính toán trọng lực, quỹ đạo chuyển động, quỹ đạo rơi.
4. Đồ hoạ 3 chiều chủ yếu thiết kế các vật thể (object), và thiết kế hộp không gian (sky cube).
5. Tìm hiểu về cinematic cho game, tức là việc chọn góc nhìn, tầm nhìn, điểm đặt máy quay v...v....


2 và 3 bạn sẽ không cần quan tâm nếu bạn dùng các game engine. Nhưng nếu bạn lập trình với thư viện OpenGL hoặc DirectX thì cần quan tâm nhiều.

1 Nếu dùng game engine thì sẽ có các ngôn ngử đặc thù của nó kiểu như lite-C, cú pháp khá giống C và C++. Nên bạn phải học tốt C/C++.

Game không phải là công việc của một người đâu, bạn cần có một nhóm và phân chia công việc rõ ràng.

http://www.gamedev.net/
