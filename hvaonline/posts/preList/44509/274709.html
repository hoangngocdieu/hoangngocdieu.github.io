Hi bạn,
Việc tạo user bạn có thể dùng lệnh SQL được mà - bạn kéo phía cuối link http://www.pixelinx.com/2010/10/creating-a-mail-server-on-ubuntu-using-postfix-courier-ssltls-spamassassin-clamav-and-amavis/ có comment của "Kozley" hỏi về vấn đề tạo thêm nhiều user đó.
Nếu bạn muốn tạo user thông qua web thì ban tham khảo thêm "postfixadmin".
Nếu muốn dùng webmail thì bạn có thể xài "squirrelmail" hoặc "roundcube" ...
Còn về vấn đề spam/virus thì thấy trong link bạn đưa có chỉ cấu hình "spamassassin-clamav-and-amavis" rồi.
Tài liệu bạn search google nhé.
Có chỗ nào sai sót thì thông cảm.

Hix... mình chuyển qua test với domain thật ko dùng DNS local test nữa, mình lấy 1 domain trên godaddy, mà máy mail server cài y chang thì nó test telnet smpt send mail bằng domain godaddy thì lỗi 550 authentication required !!! mình mò hoài ko biết lỗi ở đâu nữa, dùng DNS local thì phân giải ok mà domain thật thì vậy, cấu hình godaddy mình thêm A record - trỏ tới ip thật của mình đang làm mailserver trên máy ảo , còn CNAME mail mặc định godaddy thì mình xóa. MX record thì mình giữ nguyên, mà nó vẫn báo lỗi authentication, test trên Thunderbird thì báo sai user or pass nhưng mình làm y như local ko sai gì hết Thật là mệt, mình chỉ lo godaddy cấu hình ko biết có sai gì ko nữa.
