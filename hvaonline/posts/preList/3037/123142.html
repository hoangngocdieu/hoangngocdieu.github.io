Hèm, Khoai lại chưa hề dùng liloconfig chứ.
Bạn hãy boot backtrack livecd. Sau đó tạo một file có nội dung như sau:
Code:
# File config cho lilo

boot = /dev/hda
# Thay hda bằng sda hoặc hdx, sdx sao cho phù hợp

prompt
timeout = 50
# Timeout trong 5 giây

# Backtrack boot config
image = /boot/vmlinuz # Bạn chỉnh lại path đến kernel đang dùng
root = /dev/hda5  # Tương tự, bạn chỉnh lại partition cho chính xác
                       # Khoai chỉ đoán hda3 dựa vào bài post của bạn
label = backtrack
read-only
# Kết thúc option cho boot vào backtrack

# Giờ đến boot cho Windows
other = /dev/hda1      # Chỉnh lại partition cho phù hợp với Win
label = Windows
# Kết thúc option cho boot vào Win


Save file text đó lại với tên bất kỳ. Ở đây Khoai đoán partition của bạn như sau:
Code:
hda1-------<hda3-------hda4--------hda5------hda6>
Win       /boot      /home        /       <swap>

Nếu không phải bạn thay đổi file config lại cho thích hợp
Sau đó bạn run
Code:
lilo -C /tmp/lilo.conf -M /dev/hda

Khoai giả sử bạn đã lưu file text config tại /tmp/lilo.conf và HDD của bạn ở device /dev/hda. Bạn có thể thay đổi lại 2 thông số trên cho phù hợp. Sau đó bạn hãy cho biết out-put của lilo như thế nào. Nếu output có dạng
Code:
LILO version.....
....
....

Boot image: /boot/vmlinuz
Added backtrack *

Boot other: /dev/hda1, on /dev/hda, loader CHAIN
Added Windows

Writing boot sector.

thì việc install lilo đã thành công. Bạn có thể reboot lại máy và chọn Windows hay backtrack. Nếu có error. waring, vân vân, bạn hãy log lại và post lên.

hoangclinh,

Lệnh mount là để mount một partition vào một directory. Ví dụ, bạn có một thư mục: /data, và bạn muốn sử dụng /dev/sda1 cho data thì bạn phải mount /dev/sda1 vào /data. Nếu bạn mount /dev/sda2 vào /data thì từ nay về sau, bạn truy cập /data thì bạn sẽ thấy hoặc ghi dữ liệu vào /dev/sda2, không phải là /dev/sda1 nữa. Xem thêm Code:
$ man mount
để biết các options, và để biết mount -v làm gì.

dmesg sẽ liệt kê thông tin kernel nhận ra từ hệ thống của bạn lúc bạn mới mở máy. Thông tin này thường bao gồm các phần cứng mà kernel nhận ra.

linux kernel phân biệt sda và hda tùy vào cổng giao tiếp của ổ cứng là serial hay parallel. Nếu bạn có IDE harddrive thì kernel sẽ nhận ra ổ đầu tiên là /dev/hda, ổ thứ hai là /dev/hdb. hda0 là partition đầu tiên của hda.

Tương tự cho sda, nhưng bạn phải sử dụng SATA, hoặc SCSI, hoặc phải có trình simulation cho PATA để kernel nhận ra ổ cứng là sda.
