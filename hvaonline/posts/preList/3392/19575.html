Đậy là 1 example về cấu hình dhcp for fedora:
Kiểm tra package dhcp-<version>.i386.rpm
# rpm -qa | grep dhcp
Nếu không có thì install package dhcp-<version>.i386.rpm
# rpm -ivh dhcp-<version>.i386.rpm
File cấu hình dhcp /etc/dhcpd.conf nếu không có thì copy file ví dụ của cấu hình dhcp /usr/share/doc/dhcp-<version>.conf.sample
# cp /usr/share/doc/dhcp-<version>.conf.sample /etc/dhcpd.conf
Chỉnh sửa file cấu hình dhcpd.conf:
Code:
ddns-update-style interim
ignore client-updates
 
subnet 192.168.1.0 netmask 255.255.255.0 {
 
   # The range of IP addresses the server
   # will issue to DHCP enabled PC clients
   # booting up on the network
 
   range 192.168.1.201 192.168.1.220;
 
   # Set the amount of time in seconds that
   # a client may keep the IP address

  default-lease-time 86400;
  max-lease-time 86400;
 
   # Set the default gateway to be used by
   # the PC clients
 
   option routers 192.168.1.1;
   # Don't forward DHCP requests from this
   # NIC interface to any other NIC
   # interfaces
 
   option ip-forwarding off;
 
   # Set the broadcast address and subnet mask
   # to be used by the DHCP clients

  option broadcast-address 192.168.1.255;
  option subnet-mask 255.255.255.0;
}

Ngoài ra command để xem hướng dẫn của các tuỳ chọn là:
# man dhcp-options
Cấu hình hệ thống để bật dịch vụ dhcpd khi khởi động máy:
# chkconfig dhcpd on
Khởi động dịch vụ dhcpd:
# service dhcpd start

Trong một vùng địa chỉ mạng tốt nhất chỉ nên có một cái DHCP server (tránh đụng độ IP và những thứ khác không cần thiết) nếu không hiểu kỹ về cấu hình DHCP. DHCP server trên linux và trên windows có cùng một chức năng là cấp IP động không phân biệt hệ thống này hay hệ thống khác nên có thể cấp IP cho bất kỳ PC nào có kết nối mạng với DHCP server.
