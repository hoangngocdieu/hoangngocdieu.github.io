Đồng chí cho tôi hỏi đ/c phân vùng USB theo định dạng FAT, FAT32 hay NTFS?
Khi bạn thực hiện theo hướng dẫn tạo file autorun bạn có gán cho nó thuộc tính hiden system không? Bạn có mở tính năng thấy được các file System chưa? Nếu đã thấy được file autorun rồi thì:

Nếu định dạng theo NTFS mà có sử dụng lệnh “cacls E:\autorun.inf /D Everyone” hoặc đã phân quyền hệ thống file autorun không cho 01 user nào hay bất kỳ ai truy cập, thay đổi thì bạn hãy sử dụng máy nào đã format cái USB đó hoặc mang USB đó đến máy nào được cài HDH trên phân vùng NTFS vào acc có quyền Admin để lấy lại quyền sở hữu file autorun đó, sau đó phân quyền Toàn quyền cho Everyone là sử dụng lại đựơc file autorun bình thừong thôi. Hình như Linux cũng rất tôn trọng cái phân quyền hệ thống của phân vùng được định dạng NTFS thì phải?

Nếu USB được phân vùng theo định dạng FAT, FAT32 hoặc định dạng NTFS nhưng không phân quyền hệ thống file autorun thì bạn kiểm tra xem file autorun của bạn có thuộc tính readonly không, nếu có thì bỏ chọn thuộc tính đó đi rồi mở file autorun đó ra thêm dòng “icon=tên của icon” rồi save là, hoặc tạo file autorun khác chép đè lên là được.

File autorun không cho sửa chữa có thể file autorun nằm trên phân vùng được định dạng NTFS và đã phân quyền hệ thống file đó hoặc đã bị nhiễm virus đang chạy.
Lệnh Cacls được sử dụng trên phân vùng NTFS(hoặc HDH được cài trên phân vùng NTFS), bạn vào Start->Run-> gõ CMD(1 dạng củaCommand Prompt).
