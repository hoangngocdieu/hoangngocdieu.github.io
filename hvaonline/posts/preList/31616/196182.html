+Bắt tay vào hành động , biến cái "no" thành cái "yes" để compile
Những việc đã làm :
_ Cài đặt X11 lib với lệnh : yum groupinstall "X Software Development" -y
_Cài gcc-c++

[root@MinhTuan xvnkb-0.2.9a]# rpm -qa | grep gcc-c++

_Cài đặt vài thứ linh tinh gì đó chả nhớ

Và ...

_ Công việc tiếp theo đã làm :
+ Bắt tay vào giải quyết Checking uchar...no bằng cách gõ :
#yum search uchar
Kết quả đây :
gucharmap.i586 : Unicode character picker and font browser
gucharmap.x86_64 : Unicode character picker and font browser
gucharmap-devel.i586 : Libraries and headers for libgucharmap
gucharmap-devel.x86_64 : Libraries and headers for libgucharmap

Mình đã ngắm kỹ 2 cô em gucharmap.x86_64 và gucharmap-devel.x86_64( vì mình dùng bản FC11x86_64)
+Tiếp tục mình dùng :
#yum install gucharmap-devel.x86_64
#yum install gucharmap.x86_64
Kiểm tra lại bằng : rpm -qa | grep gucharmap
Thì gucharmap đã được cài :

[root@MinhTuan xvnkb-0.2.9a]# rpm -qa | grep gucharmap
gucharmap-devel-2.26.2-1.fc11.x86_64
gucharmap-2.26.2-1.fc11.x86_64

Tôi cài đặt scim-unikey như sau trên Fedora 11
Code:
yum install scim scim-libs scim-bridge scim-bridge-gtk


Trang chủ :
http://code.google.com/p/scim-unikey

Download
http://scim-unikey.googlecode.com/files/scim-unikey-0.3_r32-1.fc11.i386.rpm

Sau khi cài đặt xong, mở System - Preferences chọn Input Method

Đánh dấu vào mục Enable input method feature sau đó chọn Scim .
Tiếp theo logout tài khoản để bắt đầu sử dụng scim-unikey

Good luck

Đây là biến của nó mà mình vẫn đang dùng trên ubuntu.
export XMODIFIERS="@im=xvnkb"
export XIM=xvnkb
export GTK_IM_MODULE=xim
export QT_IM_MODULE=xim
export LD_PRELOAD=/usr/lib/xvnkb/xvnkb.so.0.2.9a
