- JDK bạn có thể vào sun.com để download, các bước thực hiện rất dễ dàng "straight forward"
- bạn nên cài Apache webserver thay vì tomcat, vì nó là một app webserver khá phổ biến và an toàn, nhiều người có thể giúp bạn hơn.
- bạn thử đăng ký 1 tài khoản trên no-ip.com xem sao, bạn sẽ có 1 tên miền đại loại như tenban.no-ip.com, nó sẽ có luôn một IP updater cho bạn cài đặt. tuy nhiên tùy theo lựa chọn cảu bạn, tôi chỉ khuyên

- các máy trong tiệm bạn chạy ok với website của bạn mà các cái khác ngoài tiệm ko xem được là do phần IP, rất có thể bạn dùng local IP thay vì Public IP

ví dụ như 10.0.0.X hoặc 192.168.0.X thay vì 202.21.12.134

bạn nên xem lại IP của bạn. ( bạn có thể vào Start menu chọn Run đánh vào CMD rùi gõ ipconfig để xem IP của mình)

good luck

Cách hack vào router thì tui chịu, còn việc ánh xạ (forward) đến máy nào thì chỉ có cách config trên router mà thôi. Tùy theo port và/hoặc IP thì admin có thể route đến 1 máy và/hoặc port của máy đó trong LAN.
Ví dụ thông thường nhất: trong LAN có 1 server, thông thường server này cũng kiêm lun webserver, lúc đó admin thường config mỗi khi có request đến port 80 của router sẽ forward đến server này. Vì vậy khi bạn đứng trong LAN và dùng domain trỏ đến Public IP (của router) bạn sẽ "đc" route đến server mình vừa nói.
