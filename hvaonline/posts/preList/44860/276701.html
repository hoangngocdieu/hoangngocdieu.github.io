Dear all,

Hiện nay, cty có sử dụng email postfix và dovecot sử dụng IMAP dùng webmail là squirremail. Cách đây một tháng có hiện tượng email thi thoảng vào rất chậm và không xác thực được khi kết nối bằng outlook và thường thì phải stop dịch vụ dovecot và start lại .Tôi xem log emailerror thì có lỗi sau

Jun 18 09:17:05 postfix01 dovecot: master: Error: service(auth-worker): Socket already exists: /var/run/dovecot/auth-worker
Jun 18 09:17:05 postfix01 dovecot: master: Error: service(anvil): Socket already exists: /var/run/dovecot/anvil
Jun 18 09:17:05 postfix01 dovecot: master: Error: service(anvil): Socket already exists: /var/run/dovecot/anvil-auth-penalty
Jun 18 09:17:05 postfix01 dovecot: master: Fatal: Failed to start listeners


Cấu hình dovecot của tôi như sau:

protocols = imap
listen = *
mail_location = maildir:~/Maildir
disable_plaintext_auth = no

auth default {
socket listen {
client {
path = /var/spool/postfix/private/auth
mode = 0660
user = postfix
group = postfix
auth_cache_size = 4096
auth_cache_ttl = 14400
}
}
mechanisms = plain login
}

Tôi sử dụng chứng thực StartTLS cho imap và smtp

Nếu ai đã gặp lỗi như tôi mong được chia sẻ.Cty tôi có khoảng 180 nhân viên

Mình có thử tra qua trên google thì thấy có 1 link : https://bugs.launchpad.net/ubuntu/+source/dovecot/+bug/873390
Bug này là do parent process đã kill nhưng child process thì không nên khi start lên sẽ bị tranh chấp port hoặc socket do các child process trước đó chưa bị kill đang nắm giữ.
Cách giải quyết chắc là phải update dovecot thôi.
