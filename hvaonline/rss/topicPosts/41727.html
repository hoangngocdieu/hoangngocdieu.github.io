Muốn tự cười vào mặt mình à. Đọc cho kỹ rồi hẵng cười. Tới nói chử kí đó là một file hình, file hình đó là kết quả của một quá trình xử lý của 1 file script. Việc tạo ra file hình sữ dụng hàm: Code:
imagecreate ( int $width , int $height );
Xem ở đây biết chi tiết. http://php.net/manual/en/function.imagecreate.php Tớ đã reply cái bài này chừng này lần rồi, thì bạn cũng cố gắng hiểu giùm tớ. Tớ nói rõ và chậm rãi: Bạn vào một trang web, hoặc diễn đàn, thấy một thành viên có chữ kí là một cái hình trong đó chứa thông tin là IP của bạn kèm theo tên trình duyệt bạn đang dùng cũng như, hệ điều hành bạn dang dùng, ngoài ra còn có thể hiện thi location của bạn kèm theo một số thông tin theo như bạn là nhạy cảm và làm bạn hoảng sợ không có căn cứ, nên bạn lên đây lập topic hỏi. Nhưng vấn đề của bạn như sau. Khi bạn xem hình của người đó bạn thực hiện một HTTP Request tới nơi chứa hình, vì khi hình chèn vào chử kí, vô tình request này sẽ chứa các thông tin mà theo như bạn là nhạy cảm bao gồm IP, User-agent,....Người ta sẽ dùng mod-rewrite để làm cho nó giống với file hình nhưng thực chất là xữ lý bằng 1 file PHP. Các thành phần của biến $_SERVER[] sẽ được sử dụng để đọc IP và User-agent: Code:
$_SERVER['REMOTE_ADDR']; //The IP address from which the user is viewing the current page.
 $_SERVER[''HTTP_USER_AGENT']//User-agent
Kết quả sẽ được hiển thị bằng : Code:
imagecreate ( int $width , int $height );
