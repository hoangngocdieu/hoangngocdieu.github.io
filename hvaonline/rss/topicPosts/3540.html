Discovering Computers and Deploying Agents Tới phần này, chúng ta coi như đã cài xong MOM 2005 Database, MOM 2005 Management Server và consoles. Với các phần cơ bản đã cài đặt, bước tiếp theo chúng ta sẽ cài đặt agent trên các máy tính sẽ được quản lý. Nên nhớ rằng, trong hầu hết các tổ chức, không phải tất cả các máy đều được giám sát và quản lý bởi MOM 2005 bởi còn bị phụ thuộc vào tài nguyên. Ràng buộc tài nguyên ở đây có thể là do khả năng xử lý của MOM 2005, giới hạn của network và giới hạn của số nhân sự để quản lý. MOM 2005 có khả năng search mạng cực mạnh và có thể tìm tất cả các máy tính. Administrators có thể nhận dạng những máy tính sẽ được giám sát và quản lý bởi MOM 2005, sau đó admin có thể triển khai agent. Một tính năng khá hay ở MOM 2005 được mô tả như hình dưới. Nếu trong một tổ chức có 2 Management, 2 nửa số máy tính được quản lý được cấu hình để sử dụng hai Management Server như là máy chủ chính và mỗi nửa này sẽ sử dụng Management Server kia như là máy chủ thứ hai.
Discovering Computers Bước tiếp theo, ta phải cấu hình discovery rules để MOM 2005 có thể tìm ra các máy tính để quản lý. Computer Discovery rules đơn giản như việc chỉ định tên máy tính hoặc tạo các chỉ định tìm kiếm. Sử dụng thông tin tìm kiếm tùy biến cho phép khả năng linh hoạt cao và có khả năng tìm kiếm các máy tính trong mạng sử dụng nhiều Management Servers. Một chú ý quan trọng là không tạo các rule giống nhau cho các Management Server trong cùng một Management Group. Nếu một máy tính được discover bởi nhiều Management Server, nó sẽ tự động thay đổi primary server mỗi lần một Management Server nào đó thực hiện công việc discover định kỳ. Lấy ví dụ, nếu hai Management Servers cùng được cấu hình discover rule để tìm domain controller thì Management Server đều tiên sẽ tự động chạy và triển khai agent của nó, sau đó Management Server thứ hai sẽ triển khai agent của mình tương tự như vậy trong lần discover định kỳ của mình và gây ra hiện tượng thay đôi primary server của các máy tính. Để tạo một Computer Discovery Rule, ta thực hiện các bước sau: 1. Trên MOM 2005 Administrator Console, mở Administrator Console, bấm vào dấu cộng đằng trước thư mục Administration và thư mục Computers để mở rộng cây thư mục như hình dưới:
2. Bấm phải chuột vào Computer Discovery Rules và chọn Create Computer Discovery Rule như hình sau:
Chú ý: Computer Discovery Rules đã có các rule mặc định. Việc thêm các rule sẽ được liệt kê phía dưới rule mặc định sau khi chúng được tạo. Các rule mặc định cho Management Servers không thể xóa. 3. Sử dụng drop-down box để chọn Management Server nào mà rule sẽ sử dụng. 4. Sử dụng drop-down box để chọn Rule type là Include hoặc Exclude. 5. Gõ vào tên Domain hoặc sử dụng NetBIOS domain name. 6. Trong phần Computer name, lựa chọn các option so khớp và các ký tự để so khớp. Chú ý: Việc đặt tên cho các máy tính theo chuẩn với các tiếp đầu ngữ hoặc tiền tố là một phần quan trọng. Lấy ví dụ, nếu tất cả các web server đều đặt chung một tiếp đầu ngữ (mail..., web...) thì rule sẽ chỉ discover các server này mà thôi. MOM 2005 chỉ hỗ trợ các ký tự English cho cách đặt tên máy. 7. Sử dụng drop-down box để chọn loại máy tính là: Servers, Clients, hay Servers and Clients. 8. Tick vào box Apply query criteria để sử dụng truy vấn chuẩn cho tùy chọn domain controllers. Hình dưới là một ví dụ của việc discovery tất cả các server trong domain và cấu hình để chúng sử dụng Management Server có tên là Bester. Sử dụng ký tự đại diện * có thể discover tất cả các máy tính, nhưng nếu sử dụng trong một tổ chức nhỏ thì đây không phải là một vấn đề. Với một tổ chức lớn, ta có thể sử dụng tiếp đầu ngữ để discover các Group mà ta mong muốn.
Sau khi cài đặt một Management Server mới và cấu hình Computer Discovery Rules thì vẫn sẽ không có động tĩnh gì cho đến thời điểm discovery định kỳ. Để tiến hành ngay quá trình discover, ta có thể làm theo bước sau: 1. Mở MOM 2005 Administrator Console, mở rộng thư mục Administration và thư mục Computers, click vào Management Servers. 2. Trong bảng bên phải, nhấn phải chuột vào một Management Server và chọn Run Computer Discovery Now. Use the Install/Uninstall Agent Wizard Cũng như tất cả các hệ điều hành và ứng dựng của Microsoft, ta có rất nhiều cách để đạt được cùng một mục đích nhưng có một phương thức nhanh và dễ dàng cho việc cài đặt agent cho các máy tính trong mạng là sử dụng Install/Uninstall Agents Wizard. 1. Trên máy cài MOM 2005 Administrator Console, mở Administrator Console, mở rộng thư mục Administration và thư mục Computers. 2. Click vào Action menu item trên cùng và chọn Install/Uninstall Agents Wizard. 3. In the Install/Uninstall Agents Wizard welcome screen, click Next. 4. Trong cửa sổ Install or Uninstall Agents, chọn nút Install Agents và click Next. 5. Trong cửa sổ Method for Discovering Computers and Installing Agents, chọn nút Browse for or type in specific computer namesn. Chú ý: Tại đây có một tùy chọn khác là chọn Search criteria. Nhiều trường hợp sử dụng tùy chọn Search criteria bởi vì nó linh hoạt và mạnh mẽ để chỉ định ra các máy tính thích hợp cho việc cài đặt agent khi có một lượng lớn máy tính. Để sử dụng tùy chọn này ta phải Computer Discovery Rules. Tất cả các rule sử dụng cho wizard này đều là Include rule và tất cả rule này sẽ cài đặt kiểu quản lý là Agent-managed. 6. Trong cửa sổ Computer Names, click và nút Browse và gõ vào tên máy tính, có thể là NetBIOS name hoặc DNS name rồi click OK. 7. Kiểm tra lại danh sách tên các máy rồi click Next 8. Tại cửa sổ Agent Installation Permissions, chọn nút Management Server Action Account và click Next. 9. Tại cửa sổ Agent Action Account, chọn nút Local System và click Next. 10. Tại cửa sổ Agent Installation Directory, kiểm tra lại đường dẫn cài đặt rồi click Next. 11. Tại cửa sổ Completing the Install/Uninstall Agents Wizard, ta có thể xem lại tất cả các lựa chọn đã cài đặt. Click Finish. Chú ý: Để MOM 2005 agents có thể gửi báo cáo về Management Server, ta phải mở các port sau: o — RPC Endmapper Port: 135 o — Server Messenger Block Port: 455 o — MOM Port: 1270 (both UDP and TCP) Firewall mặc định có trong các hệ điều hành Windows Server 2003 Service Pack 1 và Windows XP Professional Service Pack 2 đều tự động block các port này. Configuring Agentless Managed Computers Bởi vì không phải tất cả các máy tính được quản lý đều dùng agent nên MOM 2005 hỗ trợ cho việc quản lý máy tính không cài agent (agentless). Các máy tính được discover bởi Management Server mà không thể cài agent (vì các lý do đã nêu ở trên) đều có mặt tại thư mục Unmanaged Computers như hình dưới:
Để cấu hình một máy tính chưa được quản lý thành một máy tính được quản lý mà không có agent, ta thực hiện các bước sau: 1. Mở MOM 2005 Administrator Console, mở rộng thư mục Administration và thư mục Computersr, click vào Unmanaged Computers. 2. Nhấn phải chuột vào máy tính tại cửa sổ bên phải và chọn Start Agentless Management như hình dưới:
3. Click vào thư mục Pending Actions. 4. Nhấn phải chuột vào máy tính và chọn Start Agentless Management Now. Cài đặt MOM 2005 Web Console MOM 2005 cung cấp khả năng cài đặt Web Console trên một máy tính độc lập. Máy tính đó có thể sử dụng MOM 2005 để cài đặt hoặc gỡ bỏ thủ công agent từ các máy tính đang được quản lý. 1. Log on vào server với quyền Admin. 2. Sau khi chạy chương trình cài đặt, nhấn vào đường dẫn Check Prerequisites để tiến hành kiểm tra các thành phần cần thiết khi cài đặt MOM 2005 Web Console. 3. Sau quá trình kiểm tra và không có vấn đề gì, ta nhấn vào đường dẫn Install Microsoft Operations Manager 2005. 4. Click Next trong cửa sổ Welcome to the Microsoft Operations Manager 2005 Setup Wizard. 5. Chấp nhận license agreement và click Next. 6. Nhập vào User Name, Organization Name, CD Key, rồi click Next. 7. Chọn nút Custom rồi click Next. Chú ý: Nếu ta chọn kiểu cài đặt là Typical thì quá trình cài đặt sẽ cài đặt cả Database, Management Server lẫn consoles. Tùy chọn Custom ta chọn chỉ để cài đặt Web Console. 8. Xóa MOM 2005 Database, MOM 2005 Management Server, và MOM 2005 User Interfaces. Enable tùy chọn MOM 2005 Web Console và click Next như hình dưới.
9. Click Next trong cửa sổ Prerequisite Check Passed. 10. Chọn port number của MOM Database Server Instance và click Next. 11. Điền các thông tin vào trong box Enter the Management Server Action Account và click Next. 12. Điền thông tin vào box Enter the Data Access Server Account như User Account, Password, và Domain hoặc thông tin máy tính local rồi click Next. 13. Click Install để cài đặt MOM 2005 Web Console. 14. Click Finish để hoàn tất quá trình cài đặt MOM 2005 Web Console. Sau khi cài đạt Web Console, ta có thể truy cập vào qua đường dẫn http://www.servername:1272 Cài đặt MOM 2005 Reporting MOM 2005 Reporting là một tùy chọn của MOM 2005. Mặc dù nó không cần thiết lắm nhưng cũng có nhiều tổ chức sử dụng nó để tạo các báo cáo từ dữ liệu chứa trong cơ sở dữ liệuS MOM 2005 Reporting. MOM 2005 Reporting yêu cầu: SQL Server 2000, SP3a hoặc cao hơn. - SQL Server 2000 Reporting Services, SP2 hoặc cao hơn - MSSQLSERVER và SQLSERVERAGENT được đặt tự chạy khi máy tính khởi động. - Schedule Task Account Chú ý: Schedule Task Account được sử dụng để cung cấp quyền thực hiện công việc truyền dữ liệu từ MOM 2005 Database tới MOM 2005 Reporting database. Account này cần phải trở thành một domain account nếu như MOM 2005 Reporting database nằm khác Server với MOM 2005 Database. - Data Transfer Task Account - Reporting User Account Reporting User Account được sử dụng để cấp dữ liệu an toàn cho MOM 2005 Reporting server để kết nối với MOM 2005 Reporting database. - Visual Studio .NET 2003 to create new reports Chú ý: Visual Studio .NET 2003 không cần thiết khi ta chỉ xem báo cáo nhưng bắt buộc phải cài đặt nếu như ta muốn thay đổi, sửa chữa hoặc lựa chọn các báo cáo. Chọn Server cho việc cài đặt MOM 2005 Reporting MOM 2005 Reporting có thể được cài đặt trên cùng server với SQL Server 2000 giống như MOM 2005 Database. Hiệu năng sử dụng có thể thấp xuống khi tất cả cơ sở dữ liệu cùng được cài đặt trên cùng một SQL server, khi thực thi các báo cáo, khi chạy backup, và cả khi MOM 2005 Database quá bận khi xử lý các dữ liệu. Nếu hiệu năng của SQL server không phải là một vấn đề thì ta sẽ tiết kiệm được một đống tiền khi ta để tất cả databases trên cùng một server. MOM 2005 Reporting cũng có thể cài đặt trên cùng server với Management Server. Nếu ta cài đặt MOM 2005 Reporting trên cùng với server với Management Server, hãy chắc chắc là nó không cài chung vào cùng thư mục với Management Server. Nếu cài chung thư mục thì quá trình cài đặt sẽ xẩy ra lỗi. Sau đây là các bước cài đặt: 1. Log on vào computer nào ta định cài đặt MOM 2005 Reporting với quyền Admin và chạy chương trình cài đặt MOM 200. 2. Sau khi khởi động chương trình cài đặt, ta tiến hành kiểm tra các yêu cầu cần thiết bằng cách nhấn vào đường dẫn Check Prerequisites và chọn nút MOM 2005 Reporting. 3. Sau khi quá trình kiểm tra hoàn tất và không có vấn đề gì, nhấn vào đường dẫn Install Microsoft Operations Manager 2005 Reportin. 4. Click Next tại cửa sổ Welcome to the Microsoft Operations Manager 2005 Reporting Setup Wizard. 5. Chấp nhận license agreement. 6. Nhập vào User Name và Organization Name. 7. Kiểm tra lại thư mục cài đặt rồi click Next. Đường dẫn này có thể thay đổi 8. Nhập vào tên của SQL Server Reporting Services Server Name nếu như tên server khác với tên mà chương trình tự động nhập rồi click Next. 9. Click Next tại cửa sổ Prerequisite Check Passed. 10. Chọn port number của MOM Database Server Instance và click Next. 11. Enter the MOM 2005 Database Server information and click Next. 12. Use the drop-down box in the SQL Server Database Instance window to select the proper SQL instance and click Next. 13. In the Database and Log Files Information window, enter the database size for the Reporting database; then use the Advanced button and the two Change buttons in the Advanced window to move the database and log files to the best locations for performance and redundancy. Click OK, and then click Next. Note: This database can be configured to Automatically grow file (once it is installed), unlike the MOM 2005 Database, because the maximum size of the Reporting database is not constrained to 30GB in size. 14. Nhập vào Data Transfer Task Account và click Next. 15. Nhập vào Reporting User Account và click Next. 16. Chọn nút Yes, I want to send operational reports (Recommended) rồi click Next. 17. Click Install để cài đặt MOM 2005 Reporting. 18. Click Finish để hoàn tất quá trình cài đặt MOM 2005 Reporting. Sau khi cài đặt, ta sẽ thấy chương trình như hình dưới.
