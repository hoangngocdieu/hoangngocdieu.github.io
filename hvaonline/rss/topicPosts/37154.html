Không nhất thiết phải cài đặt mới chạy được. Mấy chương trình nho nhỏ copy và chạy vù vù như thường nếu nó ko đụng tới System (write registry, write system folder...). Còn như mình đảm nhiệm phần tạo Setup cho các soft thì thấy file Setup nó có 1 số tính năng hơn với việc copy và paste. 1. Bootstrapper cho các Requirements (.net framework, sqlexpress, windows installer...) 2. Version management (remove version cũ, chống cái cũ đè cái mới, chống cài đặt nhiều version trên một máy...) 3. User Management (cài đặt dưới quyền Admin và thực thi dưới quyền User đảm bảo tính toàn vẹn của cả file cài đặt. User ko xoá, sửa... gì được hết) 4. Tự động phân phối cài đặt được trong network (sẽ ra sao nếu như lúc trước cứ copy files cho tất cả các user và C:\Program Files\... Lỡ trong máy user đó ko có mục Program Files thì toi cơm! Installer sẽ đảm bảo rằng files sẽ được cài đặt đúng nơi nó cần đến) 5. Thân thiện với người dùng. 1 file duy nhất và cứ next nếu không hiểu.
